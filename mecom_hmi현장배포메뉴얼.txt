현장에서 가장 확실하고 유지보수하기 좋은 "배치 파일(.bat)을 이용한 자동 실행 배포 방식"을 누구나 따라 할 수 있도록 매뉴얼 문서 형태로 정리해 드립니다.

이 매뉴얼을 그대로 복사하셔서 사내 위키나 배포 가이드 문서로 활용하시면 됩니다.

📖 MECOM 지열 HMI 시스템 현장 배포 매뉴얼
문서 버전: 1.0
작성일: 2026-05-07

본 문서는 본사에서 개발이 완료된 MECOM 지열 HMI 시스템(mecom_hmi 폴더)을 현장(On-site) PC에 설치하고, 사용자가 클릭 한 번으로 시스템을 구동할 수 있도록 자동화 세팅하는 방법을 설명합니다.

1. 사전 준비 (현장 PC 세팅)
현장 PC에 파이썬(Python) 환경을 구축하고 필요한 라이브러리를 설치하는 과정입니다. (최초 1회만 수행)

1-1. 파이썬(Python) 설치
현장 PC에서 인터넷 브라우저를 열고 파이썬 공식 홈페이지(python.org)에 접속하여 최신 버전의 설치 파일을 다운로드합니다.

다운로드한 설치 파일(python-3.x.x-amd64.exe)을 실행합니다.

[매우 중요] 설치 첫 화면 맨 아래에 있는 Add Python to PATH 체크박스를 반드시 체크합니다. (체크하지 않으면 자동 실행 스크립트가 작동하지 않습니다.)

Install Now를 클릭하여 설치를 완료합니다.

1-2. 필수 라이브러리 설치
현장 PC의 키보드에서 Windows 로고 키 + R을 누르고, cmd를 입력한 뒤 엔터를 쳐서 명령 프롬프트(까만 창)를 엽니다.

아래 명령어를 복사하여 명령 프롬프트에 붙여넣고 엔터를 칩니다.

DOS
pip install streamlit pandas pymodbus
설치가 완료될 때까지 대기합니다. (Successfully installed... 메시지 확인)

2. 시스템 폴더 복사 및 포트 설정
본사에서 완성된 소스 코드를 현장 PC로 옮기고, 현장 장비에 맞게 통신 설정을 맞추는 과정입니다.

2-1. 시스템 폴더 복사
완성된 mecom_hmi 폴더 전체를 USB 등에 담아 현장 PC의 원하는 경로(예: C:\mecom_hmi 또는 바탕화면)에 복사합니다.

2-2. 현장 COM 포트 확인 및 코드 수정
현장 PC에 RS-485 통신 케이블을 연결하면 본사 PC와 다른 COM 포트 번호가 할당될 수 있습니다.

현장 PC 바탕화면의 [내 PC] 아이콘을 우클릭하고 [관리] -> [장치 관리자] -> [포트(COM & LPT)] 항목을 엽니다.

연결된 RS-485 케이블(또는 USB-to-Serial 변환기)이 몇 번 포트(예: COM3, COM4)로 잡혀있는지 확인합니다.

복사한 mecom_hmi 폴더 안의 modbus_worker.py 파일을 메모장(또는 코드 에디터)으로 엽니다.

파일 상단(약 14번째 줄)의 통신 포트 번호를 현장 장치 관리자에서 확인한 번호로 수정합니다.

수정 전: PORT = 'COM6'

수정 후: PORT = 'COM3' (현장에 맞게 변경)

변경 후 반드시 저장(Ctrl + S)합니다.

3. 원클릭 자동 실행 파일(Batch) 생성
사용자가 여러 개의 까만 창을 열 필요 없이, 아이콘 하나만 더블클릭하면 백그라운드 통신과 웹 대시보드가 동시에 켜지도록 자동화 스크립트를 만듭니다.

3-1. 실행 스크립트 작성
mecom_hmi 폴더 안에 마우스 우클릭을 하고 [새로 만들기] -> [텍스트 문서]를 클릭합니다.

만들어진 텍스트 문서를 열고 아래 코드를 그대로 복사하여 붙여넣습니다.

@echo off
title MECOM 지열 모니터링 시스템
echo ==========================================
echo MECOM 시스템을 시작합니다...
echo 창을 닫으면 통신이 종료됩니다.
echo ==========================================

:: 현재 배치 파일이 있는 위치(mecom_hmi 폴더)로 경로 이동
cd /d "%~dp0"

:: 1. 백그라운드에서 Modbus 통신 워커 실행 (데이터 수집 시작)
start /b python modbus_worker.py

:: 2. 워커가 통신을 연결할 수 있도록 3초 대기
timeout /t 3 /nobreak > NUL

:: 3. Streamlit 대시보드 화면을 기본 브라우저로 띄우기
streamlit run app.py


메모장 상단 메뉴에서 [파일] -> [다른 이름으로 저장]을 누릅니다.

아래 설정에 맞춰 저장합니다.

파일 이름: start_hmi.bat (반드시 .txt가 아닌 .bat로 끝나야 합니다)

파일 형식: 모든 파일 (.)

인코딩: ANSI 또는 UTF-8

기존에 만들어둔 텍스트 문서는 삭제하셔도 좋습니다.

3-2. 바탕화면 바로 가기 아이콘 생성 (사용자 편의성)
방금 만든 start_hmi.bat 파일을 마우스 우클릭하고 [보내기] -> [바탕 화면에 바로 가기 만들기]를 클릭합니다.

바탕화면에 생성된 바로 가기 아이콘의 이름을 MECOM 모니터링 시작 등으로 알기 쉽게 바꿉니다.

(선택 사항) 아이콘을 우클릭 -> [속성] -> [아이콘 변경]을 눌러 MECOM의 회사 로고 아이콘(.ico)으로 변경하면 더욱 전문적인 프로그램처럼 보입니다.

4. (선택 사항) PC 부팅 시 자동 켜짐 설정
현장 PC가 재부팅되었을 때 사용자가 개입하지 않아도 HMI 시스템이 알아서 구동되도록 하는 설정입니다.

현장 PC 키보드에서 Windows 로고 키 + R을 누릅니다.

실행 창에 shell:startup 을 입력하고 엔터를 칩니다.

'시작프로그램' 폴더가 열립니다.

앞서 3-2 단계에서 바탕화면에 만들어둔 [MECOM 모니터링 시작] 바로 가기 아이콘을 복사(Ctrl+C)하여 이 '시작프로그램' 폴더 안에 붙여넣기(Ctrl+V) 합니다.

설정이 완료되었습니다. 이제 PC가 켜지면 자동으로 MECOM 모니터링 시스템이 실행됩니다.

[사용 시 주의사항]

시스템을 완전히 종료하려면 화면에 떠 있는 인터넷 브라우저 창과 함께 뒤에 떠 있는 까만색 명령 프롬프트(cmd) 창을 모두 닫아주셔야 합니다. (까만 창을 닫지 않으면 백그라운드 통신이 계속 돌아가며 데이터를 기록합니다.)
